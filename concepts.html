Key Concepts Page

Browser and Web Development Concepts

These concepts help explain what browsers receive, how engines process it, and why a page may behave differently across devices.

Browser

The complete application used to navigate the Web, manage tabs, store settings, and display web content.

Browser engine

The internal system that interprets web content, calculates layout, and produces the visual page.

Rendering

The process of turning HTML, CSS, images, and other resources into pixels on the screen.

HTML

The core markup language used to describe the structure and meaning of web content.

CSS

The language used to control presentation, including colors, spacing, typography, and responsive layout.

JavaScript

A programming language used to add behavior, react to events, change the DOM, and build interactive applications.

DOM

The Document Object Model is the browser's structured representation of a web document. JavaScript can read and change it.

Web standard

A shared technical specification that helps browsers and developers implement the Web consistently.

HTTP and HTTPS

Protocols used for communication between a browser and web server. HTTPS protects the connection with encryption.

URL

The address used to identify and locate a resource on the Web.

Responsive design

A design approach that allows the same page to adjust to different screen sizes and input methods.

Media query

A CSS rule that applies styles when conditions such as viewport width are met.

How the concepts connect

  1. A user enters a URL in a browser.
  2. The browser uses HTTPS to request files from a server.
  3. The browser engine parses HTML and CSS.
  4. The browser creates and displays the DOM and visual layout.
  5. JavaScript can react to events and update the DOM.
  6. Web standards help the page behave consistently in different engines.
  7. Responsive design and media queries adjust the page for the user's screen.

Standards and accessibility

Standards are not only about making a page look the same. They also help browsers expose correct meaning to assistive technologies. Semantic HTML, alternative text, keyboard access, readable color contrast, labeled form controls, and predictable navigation improve usability for more people.
